# Claim Types and Examples

This reference provides examples of different claim types and appropriate verification approaches.

## Technical Claims

### Software Versions and Features

**Example claims:**
- "TypeScript 5.0 introduced decorators to the type system"
- "Node.js 20 includes native test runner support"
- "Next.js 14 uses the App Router by default"

**Verification approach:**
1. Search for official release notes or changelogs
2. Check official documentation
3. Verify version number and feature name are correct

**Good sources:**
- Official GitHub releases
- Product documentation sites
- Official blog announcements

### Programming Language Specifications

**Example claims:**
- "Python uses garbage collection for memory management"
- "JavaScript is single-threaded"
- "Rust prevents data races at compile time"

**Verification approach:**
1. Check official language specifications
2. Consult authoritative language documentation
3. Look for academic papers or books for theoretical claims

**Good sources:**
- Language specification documents
- Official language tutorials
- Books by language creators or experts

### API and Framework Details

**Example claims:**
- "React hooks can only be called at the top level"
- "The Fetch API returns a Promise"
- "Express.js middleware has access to req, res, and next"

**Verification approach:**
1. Check official API documentation
2. Verify with code examples if needed
3. Confirm version-specific behavior

**Good sources:**
- Official API docs
- MDN Web Docs
- Framework official guides

## Statistical and Numerical Claims

### Market Data and Statistics

**Example claims:**
- "The global SaaS market was valued at $197 billion in 2023"
- "TypeScript is used by 78% of JavaScript developers"
- "GitHub has over 100 million developers"

**Verification approach:**
1. Find the original source of the statistic
2. Verify the exact number and date
3. Check methodology if available
4. Note if it's an estimate vs. exact figure

**Good sources:**
- Market research firms (Gartner, IDC)
- Official company announcements
- Government statistical agencies
- Developer surveys (Stack Overflow Survey, State of JS)

### Performance Metrics

**Example claims:**
- "Bun is 4x faster than Node.js for HTTP requests"
- "PostgreSQL handles 50,000 transactions per second"
- "This optimization reduced latency by 30%"

**Verification approach:**
1. Look for official benchmarks
2. Check benchmark methodology and conditions
3. Note hardware/configuration used
4. Verify metrics are comparable

**Good sources:**
- Official benchmark suites
- Peer-reviewed performance papers
- Transparent benchmark projects
- Company engineering blogs with methodology

### Usage and Adoption Metrics

**Example claims:**
- "React has 10 million weekly npm downloads"
- "VS Code is the most popular IDE with 50% market share"
- "Docker has 13 million developers"

**Verification approach:**
1. Check real-time metrics if available (npm, GitHub stars)
2. Find survey data for market share claims
3. Verify date of measurement
4. Check source methodology

**Good sources:**
- npm registry statistics
- Stack Overflow Developer Survey
- GitHub statistics
- JetBrains State of Developer Ecosystem

## Historical Claims

### Technology History

**Example claims:**
- "Linux was first released in 1991"
- "The World Wide Web was invented by Tim Berners-Lee"
- "Docker was released as open-source in 2013"

**Verification approach:**
1. Check multiple historical sources
2. Verify dates and names
3. Look for primary sources when possible

**Good sources:**
- Official project histories
- Computer History Museum
- Original announcements
- Well-documented timelines

### Company and Product History

**Example claims:**
- "Google was founded in 1998"
- "The iPhone was announced in January 2007"
- "Amazon Web Services launched in 2006"

**Verification approach:**
1. Check company official history
2. Verify with news archives
3. Confirm specific dates

**Good sources:**
- Company about pages
- News archives
- Official press releases
- Business histories

## Scientific Claims

### Computer Science Concepts

**Example claims:**
- "Quicksort has O(n log n) average time complexity"
- "SHA-256 produces a 256-bit hash value"
- "TCP guarantees in-order delivery of packets"

**Verification approach:**
1. Check computer science textbooks
2. Consult academic papers
3. Verify with specifications or RFCs

**Good sources:**
- Academic textbooks
- IEEE/ACM publications
- RFC documents
- Algorithm analysis papers

### Research Findings

**Example claims:**
- "A 2023 study found that AI-assisted developers code 55% faster"
- "Research shows that code reviews reduce bugs by 60%"
- "Studies indicate that 70% of security vulnerabilities are memory safety issues"

**Verification approach:**
1. Find the original research paper
2. Verify the specific finding
3. Check study methodology
4. Note sample size and limitations

**Good sources:**
- Peer-reviewed journals
- ArXiv preprints (with caution)
- Conference proceedings
- Research institution publications

## Attribution Claims

### Quotes

**Example claims:**
- "Alan Kay said 'The best way to predict the future is to invent it'"
- "Edsger Dijkstra wrote 'Simplicity is prerequisite for reliability'"
- "Donald Knuth stated 'Premature optimization is the root of all evil'"

**Verification approach:**
1. Find the original source of the quote
2. Verify exact wording
3. Check context to ensure proper attribution

**Good sources:**
- Original publications
- Quote databases with citations
- Verified interview transcripts
- Author's collected works

### Ideas and Inventions

**Example claims:**
- "The term 'technical debt' was coined by Ward Cunningham"
- "The Model-View-Controller pattern was introduced in Smalltalk"
- "The concept of 'infrastructure as code' originated with cloud computing"

**Verification approach:**
1. Research the origin of the concept
2. Find earliest documented use
3. Verify proper attribution

**Good sources:**
- Original papers introducing concepts
- Historical documentation
- Author interviews or writings
- Computer science history resources

## Legal and Policy Claims

**Example claims:**
- "React is licensed under MIT"

**Verification approach:**
1. Check project LICENSE file
2. Verify on official repository
3. Consult OSI for license details

**Good sources:**
- Project repositories
- Open Source Initiative
- Official documentation
- SPDX License List

### Regulations and Compliance

**Example claims:**
- "GDPR requires explicit consent for cookies"
- "SOC 2 compliance is required for enterprise SaaS"
- "California's CCPA applies to businesses over $25M revenue"

**Verification approach:**
1. Check official regulatory text
2. Consult legal resources
3. Verify applicability criteria

**Good sources:**
- Official government websites
- Legal compliance guides
- Regulatory body publications
- Law firm explainers (for interpretation)

## Comparison Claims

### Technology Comparisons

**Example claims:**
- "PostgreSQL is ACID-compliant while MongoDB is not"
- "GraphQL reduces over-fetching compared to REST"
- "WebAssembly runs faster than JavaScript"

**Verification approach:**
1. Verify each component of the comparison
2. Check if comparison is fair/contextual
3. Look for official specifications
4. Consider if claim is universally true or context-dependent

**Good sources:**
- Official specifications
- Objective benchmarks
- Technical papers
- Documentation for both technologies

## Edge Cases

### Partially Factual Claims

**Example:** "React is faster than Vue"

**Analysis:**
- Contains factual elements (both are frameworks)
- But "faster" is context-dependent
- Performance varies by use case

**Verification approach:**
- Mark as partially factual
- Note the subjective component
- If specific benchmarks exist, cite them with context

### Time-Sensitive Claims

**Example:** "Python 3.11 is the latest stable version"

**Analysis:**
- Factual at time of writing
- Becomes outdated quickly

**Verification approach:**
- Verify current status
- Note if claim is outdated
- Include current information in notes

### Context-Dependent Claims

**Example:** "MongoDB doesn't support joins"

**Analysis:**
- Was true in early versions
- MongoDB added `$lookup` in version 3.2 (2015)
- Still limited compared to SQL

**Verification approach:**
- Verify current state
- Note historical context if relevant
- Clarify limitations

## Summary Table

| Claim Type | Verification Priority | Typical Sources | Common Issues |
|------------|----------------------|-----------------|---------------|
| Technical specs | High | Official docs, changelogs | Version-specific behavior |
| Statistics | High | Original research, official data | Outdated data, methodology |
| Historical facts | Medium | Multiple historical sources | Conflicting dates |
| Quotes | Medium | Original publications | Misattribution, paraphrasing |
| Performance | Medium | Benchmarks with methodology | Context-dependent results |
| Opinions | Low/N/A | Not verifiable | Subjective nature |
| Comparisons | Variable | Depends on specific claims | Context and fairness |
| Current events | High | Official announcements | Rapidly changing |
